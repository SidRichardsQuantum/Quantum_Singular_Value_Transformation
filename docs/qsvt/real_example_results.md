# Real-Example Results

This page is the rendered results ledger for plots, tables, and reports
generated by notebooks in `notebooks/real_examples/`.

`RESULTS.md` remains the compact root index for all result-producing workflows.
This page is where real-example visuals and tables should be added as notebook
outputs are committed.

## Current Status

No real-example plot, table, or JSON report artefacts are currently committed.

The real-example notebooks are tracked as clean, executable workflows. Their
results are produced when the notebooks are run locally. When an output is
committed, add it to the matching notebook section below and add a compact row
to the root [Results](../../RESULTS.md) index.

## Artefact Locations

Use these paths for committed real-example outputs:

| artefact type | suggested path | example |
| --- | --- | --- |
| plots | `results/plots/real_examples/` | `01_ground_state_filtering-overlap.png` |
| tables | `results/tables/real_examples/` | `01_ground_state_filtering-summary.csv` |
| JSON reports | `results/reports/real_examples/` | `01_ground_state_filtering-report.json` |

Record package version, random seeds, backend details, and regeneration commands
when they matter for reproducibility.

## Adding Results

For each committed artefact:

1. save the file under the appropriate `results/` subdirectory,
2. replace that notebook's `n/a` row with a link to the artefact,
3. render plots or small tables directly below the ledger row when useful,
4. add a compact index row to [Results](../../RESULTS.md),
5. include regeneration commands or notebook execution notes when needed.
